### Sub-2：Vision NN 组件

- [ ] **2.1** 创建 `mlx/src/nn/vision/mod.rs`
  - 定义 `VisionEncoder` trait（`encode`, `output_dim`）
  - 导出子模块
- [ ] **2.2** 创建 `mlx/src/nn/vision/patch_embed.rs` — PatchEmbed
  - 结构体：`weight: Array [1024, 2, 16, 16, 3]`, `bias: Array [1024]`
  - `new(weight, bias, patch_size, temporal_patch_size)`
  - `forward(&self, pixel_values: &Array, stream: &Stream) -> Result<Array>`
  - 实现 Conv3D 分解：
    1. 沿 dim=1 split 权重为 `temporal_patch_size` 个 2D 核
    2. 每个 2D 核 reshape 为 `[out_ch, h*w*in_ch]`（即 `[1024, 768]`）
    3. 输入 im2col：`[B, T, H, W, 3]` → 每帧提取 16x16 patch → `[B, num_patches, 768]`
    4. matmul 每帧结果并累加
    5. 加 bias
  - 单元测试：固定输入验证输出 shape `[B, num_patches, 1024]`
- [ ] **2.3** 创建 `mlx/src/nn/vision/vision_block.rs` — VisionBlock
  - 结构体：`norm1: LayerNorm`, `attn: VisionAttention`, `norm2: LayerNorm`, `mlp: VisionMLP`
  - `VisionAttention`：
    - `qkv: Linear [3072, 1024]`（含 bias）
    - `proj: Linear [1024, 1024]`（含 bias）
    - `num_heads: usize = 16`, `head_dim: usize = 64`
    - forward：qkv 投影 → split Q/K/V → 全局注意力（非因果）→ proj
  - `VisionMLP`：
    - `fc1: Linear [4096, 1024]`（含 bias）— GELU 激活
    - `fc2: Linear [1024, 4096]`（含 bias）
    - forward：fc1 → gelu → fc2
  - `forward(&self, x: &Array, mask: Option<&Array>, stream: &Stream) -> Result<Array>`
  - 单元测试：验证输出 shape 不变 `[B, L, 1024]`
- [ ] **2.4** 在 `mlx/src/nn/activations.rs` 中新增 `gelu_tanh(x)` — GELU pytorch_tanh 近似
  - `x * 0.5 * (1 + tanh(sqrt(2/pi) * (x + 0.044715 * x^3)))`
  - 或使用 `fast::gelu_approx` 如果 mlx-c 提供
- [ ] **2.5** 创建 `mlx/src/nn/vision/spatial_merger.rs` — SpatialMerger
  - 结构体：`norm: LayerNorm`, `fc1: Linear [4096, 4096]`, `fc2: Linear [2560, 4096]`, `spatial_merge_size: usize`
  - `forward(&self, x: &Array, grid_thw: &[(usize,usize,usize)], stream: &Stream) -> Result<Array>`
    1. 根据 grid_thw 将 `[B, H*W, 1024]` reshape 为 `[B, H/2, 2, W/2, 2, 1024]`
    2. transpose 为 `[B, H/2, W/2, 2, 2, 1024]` → reshape `[B, H/2*W/2, 4*1024]`
    3. norm → fc1 → gelu → fc2
  - 单元测试：验证序列长度缩减 4 倍，输出 dim = 2560
- [ ] **2.6** 在 `mlx/src/nn/mod.rs` 中添加 `pub mod vision;`

**批次 1 验收：** `cargo build --release` 编译通过，所有新增单元测试通过。

## 批次 2：视觉编码 + 位置编码（并行）

### Sub-3：Qwen35VisionEncoder

- [ ] **3.1** 创建 `mlx/src/nn/vision/qwen35_vision.rs`
  - `Qwen35VisionEncoder` struct：
    - `patch_embed: PatchEmbed`
    - `pos_embed: Array [2304, 1024]`
    - `blocks: Vec<VisionBlock>` — 24 层
    - `merger: SpatialMerger`
  - `impl VisionEncoder for Qwen35VisionEncoder`
  - `encode()` 实现：
    1. patch_embed.forward(pixel_values)
    2. compute_position_ids(grid_thw) → gather pos_embed → add
    3. 24 层 VisionBlock
    4. merger.forward(h, grid_thw)
  - `compute_position_ids()` — 根据 grid_thw 生成 2D 位置索引
    - 对每个 (t, h, w) 组，生成 `h_indices * max_w + w_indices` 的位置序列
    - 用 `ops::take` 从 pos_embed 中 gather
- [ ] **3.2** 创建权重加载函数 `build_vision_encoder(weights: &HashMap<String, Array>, config: &VisionConfig) -> Result<Qwen35VisionEncoder>`
  - 权重前缀 `vision_tower.`
  - 加载 patch_embed、pos_embed、24 个 blocks、merger
  - 视觉塔权重全部为 BF16，使用 `Linear`（非量化 `LinearLayer`）
- [ ] **3.3** 集成测试：加载 Qwen3.5-4B-4bit 权重，构建 VisionEncoder，用随机输入验证输出 shape

### Sub-4：Embedding 注入 + M-RoPE

- [ ] **4.1** 在 `mlx/src/model/qwen35.rs` 中新增 `forward_with_embeddings()`
  - 接受 `input_embeddings: &Array [B, L, hidden]` 替代 `embed_tokens`
  - 其余逻辑与 `forward()` 相同
- [ ] **4.2** 新增 `inject_vision_embeddings()` 函数（在 `qwen35_vl.rs`）
  - 输入：`tokens: &Array [B, L]`, `text_embeddings: &Array [B, L, hidden]`, `vision_embeddings: &Array [N, hidden]`, `image_token_id: i64`
  - 逻辑：扫描 tokens 找到 `image_token_id` 位置，用 vision_embeddings 替换对应位置的 text_embeddings
  - 输出：`merged_embeddings: Array [B, L, hidden]`
- [ ] **4.3** 新增 `mlx/src/nn/mrope.rs` — M-RoPE 实现
  - `compute_mrope_position_ids(tokens, grid_thw, image_token_id, video_token_id) -> Array [3, L]`
    - 文本 token：`[pos, pos, pos]`
    - 图像 token：`[row, col, frame]`（根据 grid_thw 展开）
  - `apply_mrope(x, position_ids, mrope_section, rope_base, stream) -> Result<Array>`
    - 将 head_dim 按 `mrope_section * 2` 分 3 段
    - 每段独立计算 cos/sin 并应用旋转
    - 拼接结果
  - `apply_rope_with_positions(x_section, positions, dim, rope_base, stream) -> Result<Array>`
    - 根据 positions 数组计算 cos/sin 频率
    - 应用 rotary embedding
  - 单元测试：固定输入对比 Python 参考输出
- [ ] **4.4** 修改 `Qwen35Attention::forward_with_cache()` — 新增 `position_ids: Option<&Array>` 参数
  - `Some(pos_ids)` → 调用 `apply_mrope()`
  - `None` → 保持现有 `fast::rope()` + partial RoPE 逻辑
  - 更新所有调用点（`Qwen35DecoderLayer::forward_with_cache`）传入 `position_ids`
- [ ] **4.5** 在 `mlx/src/nn/mod.rs` 中导出 `pub mod mrope;`

**批次 2 验收：** VisionEncoder 能加载权重并输出正确 shape；M-RoPE 单元测试通过；纯文本模式无回归。

## 批次 3：模型组装 + API + E2E（顺序）

### Sub-5：Model 构建 + API 扩展

- [ ] **5.1** 新增 `VisionConfig` 到 `mlx/src/model/config.rs`
  - `VisionConfig` 结构体（11 个字段）
  - `Qwen35Config` 新增：`vision_config`, `image_token_id`, `video_token_id`, `vision_start_token_id`, `vision_end_token_id`
- [ ] **5.2** 创建 `mlx/src/model/qwen35_vl.rs` — Qwen35VLModel
  - 结构体：`text_model: Qwen35Model`, `vision_encoder: Qwen35VisionEncoder`, `media_processor: Qwen35MediaProcessor`, `image_token_id: i64`, `video_token_id: i64`
  - `forward_vlm()` 实现：
    1. 如有 media → vision_encoder.encode() → inject_vision_embeddings()
    2. compute_mrope_position_ids()
    3. text_model.forward_with_embeddings(merged, position_ids, cache)
    4. 如无 media → text_model.forward(tokens, cache)
  - 权重加载：`build_qwen35_vl(config_path, weights) -> Result<Qwen35VLModel>`
    - 复用 `qwen35::from_config_file()` 构建文本模型
    - 调用 `build_vision_encoder()` 构建视觉编码器
    - 构建 `Qwen35MediaProcessor` from VisionConfig
- [ ] **5.3** 扩展 `mlx/src/model/mod.rs`
  - `Model` 枚举新增 `Qwen35VL(Qwen35VLModel)`
  - `Model::forward()` 中 `Qwen35VL` 分发到 `text_model.forward()`
  - 新增 `Model::forward_vlm()` 方法
  - `build_model_from_file()` 中检测 `vision_config` 字段 → 构建 VLM
- [ ] **5.4** 修改 `ironmlx/src/api/types.rs` — 多模态 API 类型
  - `MessageContent` enum（`Text(String)` / `Parts(Vec<ContentPart>)`）
  - `ContentPart` enum（`Text` / `ImageUrl` / `VideoUrl`）
  - `ChatMessage.content` 改为 `MessageContent`
- [ ] **5.5** 修改 `ironmlx/src/api/models.rs` — 多模态请求处理
  - `extract_content_parts(content: &MessageContent) -> (String, Vec<MediaItem>)`
  - 文本部分传给 chat template（`CoreChatMessage` 保持 `String`）
  - 媒体部分传给 engine 的 VLM 路径
- [ ] **5.6** 修改 `ironmlx/src/engine.rs` — VLM 推理路径
  - `EngineCommand` 新增 `media: Option<Vec<ProcessedMedia>>` 字段
  - 检测到 media 时调用 `Model::forward_vlm()`
  - 无 media 时保持现有 `Model::forward()` 路径
